欢迎访问我的算法仓库

这里面包括我的算法模板和练习题目

简单的介绍一下我的学习过程吧（如果你感兴趣的话）。
我是大一结束分流进入到了计算机学院，在放暑假之前，我们学院有个夏季实训，中间有一个程序设计比赛，那个时候，我第一次接触到了算法（在此之前，我基本上就没有搞过什么编程，当时觉得编程就是一个好高大上的东西）。我在准备这个比赛的时候（大一只学过C语言，而且还是第一学期学的，到了暑假前已经忘光了。不过这个比赛也没什么，但是当时太年轻还觉得挺重要的，哈哈哈），那个时候我做到了一个博弈论的题目，我当时很好奇啊，然后上网搜了一下，看到了左神的视频，从此，一个大门，一个新世界的大门就这样被打开了（真的，你敢想，一个几乎就是零基础的人第一个学的算法竟然是博弈论，没人引路就是这样，不过在此之后就不是了）。

我在那个暑假里面开始认真学起了算法，当时的我是一个什么状况呢？只会C语言（还不熟），编程没有基础，编程软件用的是codeblocks（没有歧视它的意思哈，我现在还觉得这个软件很好用呢，只不过感觉它差点意思），完全没有规划和目标（指的是学习算法的规划）。我一开始也没想着要打比赛之类的，真的就是单纯被这些算法所吸引，或许说是被这些智慧所吸引（到现在为止，我一直觉得算法是思想，它们应该是很美妙的，但是算法竞赛中的有的算法我只能称之为技巧，可能是我的境界达不到，我觉得它们并不如其他的美妙）。那个暑假，我主要在学动态规划，左神的视频非常详细，我真的就学了两个月的动态规划，在洛谷上也练习了很多题目。当时我还以为动态规划就是占了算法竞赛的半壁江山，现在看来当时确实是坐井观天的井底之蛙。

特别提一嘴，在这期间我学了并查集算法，这是我最喜欢的算法，我觉得这是非常美妙的算法，我至今还是认为它是我学过最具有智慧的算法。

后来大二开学了，我开始用一些算法书来学习了，当然主要还是左神的视频。那个时候我可以做的是洛谷上的黄题，觉得绿题就是很难很难的题目了，虽然跟着左神练过一些绿题还有极个别的蓝题。当时绿题能不能做呢？我觉得应该是可以的，无非就是做的慢一些，需要理解题解的时间多一些，但不敢做的原因更多是恐惧吧，如果你每开一道题都做不出来，那种挫败感，是令人很崩溃的。（在练题的整个过程中）提到挫败感，我经历过好多次，五六次应该是有的，就是在图书馆刷题，一整个下午或者一天都做不到适合自己做的题目，有的题目是过于复杂了（我不太喜欢不能用几句话就概括出整道题的解题思想的题目，这不是说实现起来复杂，是连想出来就很复杂了），有的题目是超出了当时能力的范围。我做题就是按照算法标签和难度标签来练习的，一个标签下的题目找不到一道可以适合做的，适合做的之前可能就写完了，但是同一个难度标签和算法，还是有做不出来的题目，还是会让人怀疑前面自己是不是真的掌握了。然后吃饭之前回顾自己一个上午或下午干了什么事情，发现自己好像什么都没干，但却感觉很累，难免生出放弃之感。唉，只能说坚持就是胜利，熬过去了一个难关，就是一片坦途了！

大二刚开始的时候，我开始接触搜索算法了，这个真是我学习最累的算法，写题的时候，总是不知不觉会出错，代码错了一遍又一遍，就是找不到问题。客观来讲，这个算法不难，但是这个算法让我感觉特别特别心累。这个算的上是我学习的第二大类算法了，我会有这种感觉可能是一开始的自我期待过高，以为自己学了两个月的算法，怎么说也不会比一开始差了吧。但是我忽略了最开始学习时我还是热情高涨的，就算遇到难题也可以凭借这股热情挺过去，经过冷却期之后，逐渐回到了骨感的现实中来，在被泼一盆冷水，只会感觉刺骨的冰凉。学业课程还是要同步的啊，开始学习C++了，要转型了，编程软件也要改一改了，开始用vscode了。在转换之前还是挺抗拒的，觉得C+codeblocks挺好用的，经过一段适应期之后，发现C++ + vscode用起来更舒服（这个全凭个人感受了，我更喜欢vscode的界面）。

我记得我能力上一个台阶的标志性事件，就是一天晚上做出来了一道并查集正反集的问题，这是一道绿题，从此我的主攻方向就不再是黄题了。我当时学的左神的课程里，有了很多蓝题，也能跟上左神的节奏了。我英语不太好，加上只是为了提高能力而练题，所以不会主动做codeforce的题目，左神课上会出现cf上的题，都是蓝色以上的，让我形成了一个错觉，cf上的题肯定都很难，现在想想，当时还怪好笑的。
